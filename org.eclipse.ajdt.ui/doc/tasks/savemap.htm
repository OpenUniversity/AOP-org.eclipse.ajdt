Saving crosscutting information

In order to compare the crosscutting information between different versions of a project, you first need to save out the crosscutting information from the current build into a file. This crosscutting information includes details of those locations in the code for a project that are being affected by advice or inter-type declarations.

To save the crosscutting information for a project:

  1. Select the project in the Package Explorer.
  2. Right click and select AspectJ Tools > Saving Crosscutting Map... from the context menu.
  3. Enter a name for the crosscutting map file in the dialog, and click OK.

A file is then created in the project and the crosscutting information is written to it in binary form. Crosscutting map files have an extension of ".ajmap" and as many such files as required can be created for different versions or configurations of the project. These files then serve as the basis for comparison, either with each other or with the current build.
